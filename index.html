Digital Path, Connectivity, Distance & Image Segmentation

Topics: Adjacency · Paths · Distance Measures · Segmentation Techniques · Deep Learning

01Pixels, Adjacency and Paths

A digital image can be viewed as a grid of pixels, each having integer coordinates \((x,y)\). How we define "neighbors" (adjacency) controls paths, connectivity, and regions in the image.

1.1 Types of Adjacency

  • 4-adjacency: Two pixels are 4-adjacent if one can be reached from the other by moving one step up, down, left, or right — no diagonal moves.
  • 8-adjacency: Two pixels are 8-adjacent if one is in any of the eight neighboring positions around the other, including diagonals.
  • m-adjacency: A mixed definition that uses both 4- and diagonal neighbors but avoids certain diagonal connections when a 4-connected path already exists through intermediate pixels with values in the allowed set \(V\).

1.2 Digital Path

A digital path from pixel \(p\) to pixel \(q\) is a sequence \((x_0,y_0),(x_1,y_1),\dots,(x_n,y_n)\) where \((x_0,y_0)=p\), \((x_n,y_n)=q\), and each consecutive pair is adjacent under the chosen scheme. The path length is \(n\). A path starting and ending at the same pixel is a closed path.

02Connectivity, Regions & Boundaries

Let \(S\) be a subset of pixels (e.g., all pixels with value 1 in a binary image).

  • Pixels \(p\) and \(q\) in \(S\) are connected in S if there exists a path between them lying entirely within \(S\).
  • For any pixel \(p \in S\), the set of all pixels in \(S\) connected to \(p\) forms a connected component of \(S\).

2.1 Regions

A subset \(R\) is a region if it is a connected set under the chosen adjacency. Each connected component of foreground pixels forms one region.

2.2 Boundaries

  • The boundary of region \(R\) is the set of pixels in \(R\) that have at least one neighbor not in \(R\).
  • If \(R\) is the entire image, its boundary is the set of pixels in the first and last rows and columns.

2.3 Foreground and Background

  • The union \(R_u\) of all disjoint regions (none touching the image border) is the foreground.
  • Its complement \(R_u^c\) is the background.

03Distance Measures Between Pixels

Let \(p(x,y)\) and \(q(s,t)\) be two pixels. Three key measures are defined:

Euclidean
\[ D_E(p,q) = \sqrt{(x-s)^2 + (y-t)^2} \]

Standard straight-line distance from analytic geometry.

D4 / City-block
\[ D_4(p,q) = |x-s| + |y-t| \]

Minimum steps using only vertical and horizontal moves (4-adjacency). Also called Manhattan distance.

D8 / Chessboard
\[ D_8(p,q) = \max(|x-s|,\; |y-t|) \]

Minimum steps when diagonal moves are also allowed — like a king's moves in chess (8-adjacency).

3.1 Relationship to Neighbors

  • Pixels with \(D_4 = 1\) from a center are exactly its 4-neighbors.
  • Pixels with \(D_8 = 1\) from a center are exactly its 8-neighbors.
  • \(D_4\) contours form diamonds; \(D_8\) contours form squares around the center.

3.2 Dm Distance (m-distance)

The Dm distance is the length of the shortest m-path between two pixels. It depends on both geometric positions and pixel values, since adjacency is defined only for pixels whose values belong to a specified set \(V\).

Example — pixels \(p, p_1, p_2, p_3, p_4\) where \(p, p_2, p_4\) always have value 1:

04Image Segmentation – Core Concepts

Image segmentation groups similar pixels into regions and assigns each region a class label — an extension of image classification that operates at pixel level and localizes object boundaries. The output is a segment map where each pixel's value indicates its class or segment.

Types of Segmentation

Semantic
Instance
Panoptic
  • Semantic: Assigns each pixel a class (road, car, pedestrian). Objects of the same class share one label; instances are not separated.
  • Instance: Separates pixels into distinct object instances regardless of class, e.g., person 1, person 2 in a crowd.
  • Panoptic: Combines both — each instance is segmented separately and given a semantic class label. Key for self-driving systems that need full scene understanding.

05Traditional Segmentation Techniques

Method 01
Thresholding

Choose threshold \(T\). Pixels ≥ T → 1 (foreground); pixels < T → 0 (background). Simple, fast, effective when foreground/background intensities differ greatly.

Method 02
Region-based

Region growing from seed pixels; neighbors are absorbed if similar in intensity/color. Produces large homogeneous regions. Watershed algorithm handles overlapping objects.

Method 03
Edge-based

Detect edge pixels using gradient filters (Sobel, Canny). Classifies pixels as "edge" vs. "non-edge". Useful for exact boundaries and sketch-like outlines.

Method 04
Clustering

K-means represents each pixel as a feature vector (e.g., RGB) and clusters all pixels into \(k\) groups. No seed points needed; considers all pixels globally.

6.2.1 Watershed Algorithm

Uses local maxima of a distance map as seeds and grows regions from these points, enforcing that two different seeds never merge. Particularly effective when simple thresholding would incorrectly merge touching objects.

06Deep Learning-based Segmentation

Multi-channel Segment Maps

  • A deep model outputs an n-channel map for \(n\) classes; each channel is a binary/probabilistic map for one class.
  • Ground truth is a single-channel integer image where each pixel stores a class index from 0 to \(n-1\).

Encoder–Decoder Architectures (e.g. SegNet)

  • The encoder uses convolution + downsampling to compress the image into a compact bottleneck representation.
  • The decoder upsamples back to the original resolution, producing a per-pixel class map.
  • A final sigmoid / softmax constrains output values to [0, 1], interpreted as class probabilities.

07Applications

Robotics & Machine Vision

  • Robotic grasping: instance segmentation isolates objects to be picked up.
  • Recycling object picking: separate materials by shape and color.
  • Autonomous navigation: highlight obstacles, free space, and scene elements.

Medical Imaging

  • X-ray, CT scan, and dental structure segmentation for diagnosis and treatment planning.
  • Digital pathology: cell/tissue segmentation in microscopic images.
  • Surgical video annotation: labeling organs and tools in footage.

Autonomous Vehicles & Aerial Imagery

  • Panoptic segmentation for complete scene understanding in self-driving cars.
  • Satellite imagery: extract roads, buildings, water bodies, and land cover at scale.

Fall 2025 · Q1(a) 5 marks

Scenario: An image-analysis system for two applications — (i) land cover classification from satellite images, (ii) brain MRI tumor segmentation.

  • App 1 — Satellite: Large homogeneous regions, smooth boundaries, region-oriented objects, minor imprecision acceptable.
  • App 2 — MRI: Variable tumor shapes, intensity overlap with healthy tissue, irregular/ambiguous boundaries, high inter-patient variability, pixel-level accuracy critical.
Answer

Part (i) — Land Cover: Region-based Segmentation

Best choice: Region-based segmentation (specifically region growing or the Watershed algorithm).

Justification:

  • Satellite images have large, spatially coherent areas with uniform color/texture (fields, water, forest) — exactly what region growing excels at: it expands a seed by absorbing similar neighbors until a homogeneous region is formed.
  • Boundaries are smooth and continuous, so region growing will not fragment regions the way edge-based methods might on noisy satellite data.
  • Objects are region-oriented (not thin lines), meaning clustering or thresholding of the whole image is also viable, but region-based methods give the added benefit of producing spatially contiguous regions aligned with real geography.
  • Minor boundary imprecision is tolerable, which forgives the slight inaccuracies of region growing at mixed-land-cover borders.

Part (ii) — Brain MRI Tumor: Deep Learning Segmentation (CNN Encoder–Decoder)

Why traditional methods fail:

  • Thresholding fails because tumor intensity overlaps with healthy tissue — no single threshold cleanly separates them.
  • Region growing fails because tumors lack spatial homogeneity and seed selection is unreliable with ambiguous, irregular boundaries.
  • Edge detection fails because boundaries are ambiguous and noisy — gradient-based filters cannot reliably locate tumor borders.
  • Clustering (K-means) fails because handcrafted feature spaces cannot capture the high inter-patient variability in tumor appearance.

Best choice: Deep learning — CNN Encoder–Decoder (e.g., U-Net).

Justification:

  • U-Net learns hierarchical spatial features from large datasets, automatically adapting to shape, size, and intensity variations across patients.
  • Skip connections between encoder and decoder preserve fine spatial details, enabling precise pixel-level boundary delineation even at irregular borders.
  • Trained on labelled MRI data, the model learns the statistical patterns of tumors vs. healthy tissue beyond what any hand-crafted rule can capture.
  • Outputs a per-pixel probability map, directly addressing the need for pixel-level accuracy critical for clinical decisions.

Summer 2025 · Q1 4 marks

An automated inspection system for a bottle recycling plant must:

  • (a) Detect and count each individual bottle — even overlapping/partially occluded ones — and identify damaged ones.
  • (b) Differentiate between materials: plastic bottles, glass shards, metal caps, and the conveyor belt — to filter non-recyclables and monitor debris.

Identify which type(s) of segmentation are necessary.

Answer
GoalSegmentation TypeJustification
Detect & count individual bottles (overlapping/occluded), identify damaged ones Instance Segmentation Each bottle must receive its own unique mask so it can be individually counted and its shape inspected for damage. Semantic segmentation would merge overlapping bottles into one blob; instance segmentation keeps them as separate objects.
Differentiate materials: plastic, glass, metal, belt Semantic Segmentation The system needs to classify every pixel into a material category. Individual pieces of glass or metal do not need to be counted separately — only their material class matters. Semantic segmentation assigns a class label to each pixel, enabling filtering of non-recyclables and flagging of dirty regions.
Combined requirement: Panoptic Segmentation
Panoptic segmentation unifies both — it assigns each pixel a class label (semantic goal) while also separating individual object instances (instance goal). This single framework satisfies both objectives of the recycling system.
